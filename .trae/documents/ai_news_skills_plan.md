# AI新闻抓取与解读Skills实现计划

## 项目分析

当前项目是一个新闻抓取和处理系统，包含以下核心组件：

- **fetchers**: 负责从多个新闻源抓取新闻，已包含30+个新闻源
- **scheduler**: 负责定时任务调度
- **storage**: 负责存储新闻数据
- **translators**: 负责翻译新闻内容
- **validators**: 负责验证新闻可信度
- **utils**: 工具函数

## 实现目标

创建供AI调用的skills，实现以下功能：
1. 每天自动抓取所有新闻源的新闻
2. 对新闻进行分类解读：
   - 全球热点
   - 金融热点
   - 国内大事
   - 彭博新闻解读

## 实现计划

### 1. 创建Skills目录结构

在项目根目录下创建skills目录，用于存放AI调用的skills：

```
skills/
├── __init__.py
├── news_fetcher_skill.py     # 新闻抓取skill
├── news_analyzer_skill.py    # 新闻分析skill
└── utils/                    # 工具函数
    ├── __init__.py
    └── skill_helpers.py      # skill辅助函数
```

### 2. 实现新闻抓取Skill

创建 `news_fetcher_skill.py`，实现以下功能：

- 提供命令行接口供AI调用
- 调用现有的fetchers模块抓取所有新闻源的新闻
- 支持指定日期范围抓取
- 支持按新闻源类型抓取
- 输出抓取结果到指定格式

### 3. 实现新闻分析Skill

创建 `news_analyzer_skill.py`，实现以下功能：

- 提供命令行接口供AI调用
- 对抓取的新闻进行分类：
  - 全球热点：国际政治、国际关系等
  - 金融热点：经济、市场、金融等
  - 国内大事：国内政治、社会、经济等
  - 彭博新闻：专门分析彭博社的新闻
- 生成新闻摘要和解读
- 支持按分类获取解读结果
- 输出分析结果到指定格式

### 4. 实现Skill辅助函数

创建 `skill_helpers.py`，实现以下功能：

- 新闻分类辅助函数
- 新闻摘要生成辅助函数
- 数据格式化辅助函数
- 命令行参数处理辅助函数

### 5. 集成到现有系统

- 修改 `routes/api.py`，添加Skills相关的API路由（可选）
- 更新 `scheduler/jobs.py`，添加每天自动抓取新闻的任务
- 确保Skills可以通过命令行被AI系统调用
- 创建简单的shell脚本，方便AI系统调用Skills

### 6. 测试与优化

- 编写测试用例，确保Skills功能正常
- 优化新闻抓取和分析的性能
- 确保分类准确性
- 测试命令行调用方式

## 技术实现细节

### 新闻分类实现

- 使用关键词匹配和规则引擎进行初步分类
- 结合现有的validators模块进行新闻可信度验证
- 对于彭博新闻，使用专门的解析器进行深度分析

### 新闻解读实现

- 利用现有的translators模块进行多语言支持
- 生成结构化的新闻摘要，包括：
  - 新闻标题
  - 来源
  - 发布时间
  - 核心内容
  - 相关背景
  - 影响分析

### 命令行接口设计

- 新闻抓取命令：`python -m skills.news_fetcher_skill [options]`
- 新闻分析命令：`python -m skills.news_analyzer_skill [options]`
- 支持参数：
  - `--date-range`: 指定日期范围
  - `--sources`: 指定新闻源
  - `--category`: 指定分类
  - `--output`: 指定输出格式（json、text等）
  - `--help`: 显示帮助信息

## 依赖与风险

### 依赖

- 现有的fetchers模块
- 现有的translators模块
- 现有的validators模块
- 现有的scheduler模块
- argparse模块（用于命令行参数处理）

### 风险

- 新闻源API变更可能导致抓取失败
- 分类准确性可能需要不断调整和优化
- 大规模抓取可能导致IP被封禁
- 分析结果可能存在主观性

## 解决方案

- 实现错误处理和重试机制
- 定期更新分类规则和关键词
- 使用代理IP和请求限速
- 提供多维度的分析视角，减少主观性
