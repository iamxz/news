# Trae IDE 项目规范

## 1. 代码生成规范

### 1.1 代码风格

- **代码格式化**：使用 Black 进行代码格式化，配置为 `line-length = 100`
- **代码检查**：使用 Ruff 进行代码检查，遵循项目已配置的规则
- **类型提示**：使用 Python 类型提示，遵循项目的 mypy 配置
- **命名规范**：
  - 模块名：小写蛇形命名法（snake_case）
  - 类名：驼峰命名法（CamelCase）
  - 函数名：小写蛇形命名法（snake_case）
  - 变量名：小写蛇形命名法（snake_case）
  - 常量名：全大写蛇形命名法（SNAKE_CASE）

### 1.2 代码结构

- **文件结构**：遵循项目现有的目录结构
- **模块化**：每个模块应具有单一职责
- **导入顺序**：
  1. 标准库导入
  2. 第三方库导入
  3. 本地模块导入
  4. 按字母顺序排列

### 1.3 文档规范

- **文档字符串**：所有模块、类、函数应包含文档字符串
- **注释**：关键逻辑应添加注释
- **README.md**：每个目录应包含 README.md 文件，说明目录功能

## 2. 文件夹存放规范

### 2.1 源代码目录

- **src/**：存放所有源代码
  - **fetchers/**：新闻抓取器模块
  - **scheduler/**：定时任务模块
  - **storage/**：数据存储模块
  - **translators/**：翻译模块
  - **utils/**：工具函数模块

### 2.2 测试目录

- **tests/**：存放所有测试代码
  - **fetchers/**：抓取器测试
  - 其他测试文件按功能分类存放

### 2.3 文档目录

- **docs/**：存放项目文档

### 2.4 脚本目录

- **scripts/**：存放部署、开发、测试等脚本

### 2.5 模板目录

- **templates/**：存放 Web 模板文件

## 3. 工程构建规范

### 3.1 依赖管理

- **依赖文件**：
  - `pyproject.toml`：项目配置和依赖声明
  - `requirements.txt`：具体依赖版本
  - `requirements-dev.txt`：开发依赖

### 3.2 构建命令

- **安装依赖**：

  ```bash
  pip install -r requirements.txt
  pip install -r requirements-dev.txt
  ```

- **代码格式化**：

  ```bash
  black src/
  ```

- **代码检查**：

  ```bash
  ruff check src/
  ```

- **类型检查**：
  ```bash
  mypy src/
  ```

## 4. 测试方法

### 4.1 测试框架

- 使用 pytest 作为测试框架

### 4.2 测试文件命名

- 测试文件应命名为 `test_<模块名>.py`
- 例如：`test_weibo_fetcher.py` 测试微博抓取器

### 4.3 测试运行方式

- **运行所有测试**：

  ```bash
  pytest
  ```

- **运行特定测试文件**：

  ```bash
  pytest tests/fetchers/test_weibo_fetcher.py
  ```

- **运行测试并显示详细输出**：

  ```bash
  pytest -v tests/fetchers/test_weibo_fetcher.py
  ```

- **运行测试并生成覆盖率报告**：
  ```bash
  pytest --cov=src
  ```

### 4.4 测试规范

- **测试覆盖**：测试应覆盖主要功能点
- **测试独立性**：测试应独立运行，不依赖外部资源
- **测试数据**：使用模拟数据进行测试，避免不必要的网络请求
- **测试边界**：测试应包含正常情况和异常情况

## 5. Trae IDE 特定配置

### 5.1 .traerules 文件

- 项目根目录应包含 `.traerules` 文件，用于配置 Trae IDE 的行为

### 5.2 代码生成提示

- **模块生成**：当生成新模块时，应遵循项目的目录结构和命名规范
- **测试生成**：当生成测试代码时，应放在对应的测试目录中
- **依赖管理**：当添加新依赖时，应更新 `pyproject.toml` 和 `requirements.txt` 文件

### 5.3 工程构建集成

- **构建命令**：Trae IDE 应集成项目的构建命令
- **测试命令**：Trae IDE 应集成项目的测试命令
- **代码检查**：Trae IDE 应集成项目的代码检查工具

## 6. 最佳实践

1. **代码质量**：保持代码简洁、可读性强
2. **测试驱动**：优先编写测试，然后实现功能
3. **模块化**：将功能分解为小的、可测试的模块
4. **文档**：保持文档与代码同步
5. **版本控制**：遵循 Git 最佳实践，提交消息清晰明了

## 7. 总结

本规范旨在确保项目代码的一致性、可维护性和质量。Trae IDE 应遵循这些规范，生成符合项目要求的代码，并提供良好的开发体验。
