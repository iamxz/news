# 测试文件说明

## 测试目录结构

```
tests/
├── __init__.py
├── fetchers/         # 抓取器测试
│   ├── __init__.py
│   └── test_weibo_fetcher.py
└── README.md         # 测试文档
```

## 测试文件规范

### 1. 测试文件命名
- 测试文件应命名为 `test_<模块名>.py`
- 例如：`test_weibo_fetcher.py` 测试微博抓取器

### 2. 测试文件位置
- 抓取器测试文件应放在 `tests/fetchers/` 目录
- 其他测试文件应根据功能分类放在相应目录

### 3. 测试内容规范
- 测试应覆盖主要功能点
- 测试应包含正常情况和异常情况
- 测试应独立运行，不依赖外部资源

### 4. 测试运行方式

```bash
# 运行所有测试
pytest

# 运行特定测试文件
pytest tests/fetchers/test_weibo_fetcher.py

# 运行测试并显示详细输出
pytest -v tests/fetchers/test_weibo_fetcher.py
```

### 5. AI 输出规范

当使用 AI 生成测试代码时，应遵循以下规范：

1. **代码风格**：
   - 遵循项目的代码风格
   - 使用蛇形命名法（snake_case）
   - 保持代码简洁明了

2. **测试覆盖**：
   - 测试应覆盖核心功能
   - 测试应包含边界情况
   - 测试应验证错误处理

3. **依赖管理**：
   - 测试应使用项目已有的依赖
   - 不应引入新的依赖

4. **文档说明**：
   - 测试文件应包含文档字符串
   - 测试方法应说明测试目的
   - 测试结果应清晰可理解

5. **执行效率**：
   - 测试应快速执行
   - 避免不必要的网络请求
   - 使用模拟数据进行测试
